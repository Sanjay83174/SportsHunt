# Comparison: White Box vs Black Box Testing

| Feature | White Box Testing | Black Box Testing |
| :--- | :--- | :--- |
| **Definition** | Testing based on analysis of the internal structure of the component or system. | Testing, either functional or non-functional, without reference to the internal structure of the component or system. |
| **Alias** | Clear Box, Glass Box, Structural Testing. | Opaque Box, Closed Box, Behavioral Testing. |
| **Knowledge Required** | Requires knowledge of the internal code, design, and structure. | No knowledge of internal code is required; focuses on inputs and outputs. |
| **Focus** | Focuses on code efficiency, memory leaks, and logic paths. | Focuses on the functionality of the system and user experience. |
| **Performed By** | Primarily performed by Developers. | Primarily performed by QA Testers and End Users. |
| **Testing Level** | Mostly used in Unit Testing and Integration Testing. | Mostly used in System Testing and Acceptance Testing. |
| **Code Access** | Tester has full access to the code. | Tester does not have access to the code. |
